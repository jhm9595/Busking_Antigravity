# 웹사이트 보상형 동영상 광고(Rewarded Ads) 연동 가이드

## 개요
본 문서는 `busking.minibig.pw` 버스킹 플랫폼의 **'포인트 충전 모달 - 광고 보고 50P 받기'** 기능 구현을 위한 사전 검토 및 기획/설계 가이드입니다. 
PM 및 개발팀은 본 가이드를 바탕으로 구글 애드센스(Google AdSense) 보상형 광고 연동을 기획하고 구현해야 합니다.

## 1. 웹사이트 보상형 광고 지원 여부 및 연동 방식
모바일 앱(AdMob)뿐만 아니라 데스크톱/모바일 웹사이트 환경에서도 보상형 동영상 광고 송출이 지원됩니다.
해당 기능을 구현하기 위해서는 다음 중 하나의 방식을 선택하여 연동해야 합니다.

*   **Google Ad Manager (추천)**: 웹사이트용 보상형 디스플레이 및 동영상 광고 슬롯(GPT 코드) 발급 활용
*   **AdSense Offerwall (대안)**: 구글이 자체적으로 띄우는 메시지 팝업 기능 활용 (단, 시간 단위 콘텐츠 해제에 최적화되어 있어 커스텀 트리거 구현에 제약이 있을 수 있음)
*   **AdSense for H5 Games**: 미니 게임 요소가 강한 UI/UX를 보유한 서비스에 적합

## 2. 버튼 클릭 트리거 (사용자 인터랙션 연동)
사용자가 "광고 보고 50P 받기" 버튼을 누르는 즉시 광고 모달/창이 뜨도록 연동(버튼 클릭 트리거)이 가능합니다.

*   **프론트엔드 준비 상태**:
    *   현재 `src/components/common/PointChargeModal.tsx` 내에 버튼 클릭 이벤트를 수신하여 광고를 호출하고, 시청 완료 시 포인트를 지급하는 뼈대 코드(`handleWatchAd` 등)가 기작성되어 있습니다.
*   **향후 작업 사항**:
    *   구글 광고 계정 승인 완료 후 발급받는 구글 제공 JavaScript API(예: `googletag.pubads().display()`)를 해당 함수 내에 연동(호출)하기만 하면 완료됩니다.

## 3. 광고 무한 클릭(어뷰징) 방지를 위한 쿨타임 정책 설정 [중요]
사용자의 무차별적인 버튼 클릭을 통한 포인트 무단 적립을 방지하고, 구글 애드센스 측의 **'스팸/어뷰징 판단(계정 정지 위험)'**을 회피하기 위해 엄격한 쿨타임(빈도 제한) 정책이 필요합니다.

*   **권장 운영 정책 (예시)**: 
    *   **1시간당 최대 3회 시청** 허용
    *   1회 시청 완료 후 **최소 10분 대기 쿨타임** 적용
*   단기간 내 과도한 광고 노출 발생 시 구글 로직에 의해 노출 가치(eCPM)가 0으로 수렴하게 되거나, 최악의 경우 시스템 어뷰징으로 간주되어 광고 계정이 정지될 수 있으므로 정책적인 횟수/시간 제한이 필수적입니다.

## 4. 쿨타임 및 어뷰징 관리 아키텍처 (백엔드 설계 요구사항)
보안 상의 이유로 쿨타임 및 광고 시청 횟수 제한 로직은 프론트엔드(브라우저 쿠키, 로컬 스토리지 등)가 아닌 **반드시 백엔드(서버/DB)에서 중앙 통제**되어야 합니다.

*   **DB 스키마 추가 사항**:
    *   유저별 광고 시청 이력을 관리하기 위한 테이블 등 추가 구성 (예: `[유저 ID, 최근 시청 완료 시각, 금일 누적 시청 횟수 등]`)
*   **검증 로직 (API)**:
    *   프론트엔드에서 광고 재생을 요청할 때마다 백엔드는 DB를 조회하여 해당 유저의 쿨타임 충족 여부 및 금일 노출 제한 도달 여부를 선 검증해야 합니다.
    *   광고 시청 완료 콜백(Callback)을 수신한 최종 완료 시점에만 DB에 이력을 업데이트하고 포인트를 지급하는 트랜잭션 처리가 필요합니다.

## 5. 구글 보상형 광고 핵심 정책 준수 (위반 시 계정 즉시 정지)
*   **현금화 리워드 지급 금지**: 광고 시청에 대한 대가로 통화 가치를 지닌 '현금, 암호화폐, 범용 상품권/기프트카드' 등을 지급해서는 안 됩니다.
    *   **본 플랫폼의 적용 적합성**: 해당 플랫폼 내에서만 통용되고 소진되는 자체 가상 재화인 **'충전 포인트(P)'**를 지급하는 형태이므로 본 규정에 부합합니다.
*   **사용자 조작/강제 전환 유도 금지**: "사이트 운영 유지를 위해 클릭해 주세요"와 같이 사용자의 자발적 이성을 흐리거나 동정심을 유발하는 문구 활용을 엄격히 금합니다. 오직 "광고 시청 후 포인트 받기"와 같이 체리피킹 리워드 명시에 기반한 사용자 본인의 자발적 선택을 보장하는 UI/UX로 구성되어야 합니다.
