# 기능 명세서 (Feature Spec)

> 이 문서는 사이트의 모든 화면과 기능, 사용자 유형별 권한을 정리한 가이드입니다.
> 나중에 사용자 가이드, 홍보 영상, 튜토리알 제작 시 기준 자료로 활용됩니다.

---

## 사용자 유형 (User Roles)

| 유형 | 설명 | 식별 방법 |
|------|------|------------|
| **비로그인 사용자 (Guest)** | 회원가입 없이 둘러보기만 가능 | `userId = null` |
| **관객 (Audience)** | 로그인한 일반 사용자, 공연 시청 및 후원 가능 | `Profile.role = "audience"` |
| **싱어 (Singer)** | 버스커, 공연 등록/진행, 대시보드 사용 | `Singer` 테이블에 레코드 존재 + `Profile` 연결 |
| **관리자 (Admin)** | (현재 미구현) | - |

---

## 화면별 기능 상세

### 1. 랜딩 페이지 (`/`)
**접근**: 모두 (비로그인 포함)
**주요 기능**:
- 서비스 소개 (제목, 서브타이틀)
- **데모 체험 버튼**: `/api/demo` 호출 → `/auth/demo`로 폼 제출 → 데모 계정으로 자동 로그인
- **로그인/회원가입**: Clerk 모달 오픈
- **언어 변경**: 한국어/영어/중국어 지원
- **테마 변경**: 라이트/다크 모드 토글
- **Google Ad**: 페이지 하단 광고 영역

**권한**:
- 비로그인: 데모 체험, 로그인 버튼만 노출
- 로그인(싱어): "싱어 대시보드로 이동" 버튼 노출
- 로그인(관객): "공연 둘러보기" 버튼 노출

---

### 2. 공연 둘러보기 (`/explore`)
**접근**: 모두 (비로그인 포함)
**주요 기능**:
- **지도 뷰**: Leaflet 기반 버스킹 위치 표시 (Google Maps 대체제)
- **리스트 뷰**: 공연 카드 리스트 (제목, 위치, 시간, 상태)
- **필터**: 전체 / 라이브 중 / 팔로잉한 싱어
- **검색**: 공연명, 설명, 싱어명으로 검색
- **무한 스크롤**: 20개씩 페이지네이션
- **팔로우 토글**: 싱어 팔로우/언팔로우
- **라이브 공연 진입**: `/live/[id]`로 이동

**권한**:
- 비로그인: 조회만 가능, 팔로우/참여 불가
- 로그인: 팔로우, 라이브 참여 가능

---

### 3. 싱어 대시보드 (`/singer/dashboard`)
**접근**: 싱어만 (로그인 + Singer 프로필 존재)
**주요 기능**:
- **프로필 동기화**: Clerk 사용자 정보를 DB(Profile, Singer)와 동기화
- **포인트 잔액**: 현재 보유 포인트 표시
- **포인트 충전**: Stripe를 통한 포인트 구매 모달
- **공연 관리**: 등록된 공연 목록 (예정/진행중/종료)
  - 공연 상태별 필터링
  - 공연 시작 (라이브 모드 진입)
  - 공연 수정/삭제
- **곡 관리**: 레퍼토리 곡 추가/수정/삭제
- **북킹 요청**: 팬들의 공연 섭외 요청 목록
  - 요청 수락/거절
- **QR 코드**: 팬들이 스캔하여 라이브 페이지 진입
- **팬 목록**: 팔로워 리스트 확인
- **시계 위젯**: 현재 한국 시간 실시간 표시

**권한**:
- 싱어 본인만 접근 가능 (다른 사용자 접근 시 리다이렉트)
- Clerk `userId`로 소유권 확인

---

### 4. 싱어 라이브 공연 (`/singer/live?performanceId=xxx`)
**접근**: 해당 공연의 싱어만
**주요 기능**:
- **3패널 레이아웃**: 세트리스트 | 요청곡 | 채팅 (드래그 조절 가능)
- **공연 타이머**: 경과 시간 실시간 표시
- **세트리스트 관리**:
  - 곡 추가 (검색/수동 입력)
  - 드래그 앤 드롭으로 순서 변경
  - 곡 상태 변경 (대기/진행중/완료)
- **요청곡 관리**:
  - 관객 요청곡 수락/거절
  - 세트리스트로 이동
- **라이브 채팅**:
  - 채팅창 열기/닫기 (포인트 소모)
  - 실시간 메시지 수신
  - 시스템 알림 발송
- **공연 종료**: 종료 확인 모달 후 공연 상태 변경

**권한**:
- 공연 소유자(싱어)만 접근 가능
- 채팅 열기: 포인트 차감 (`usePointsForChat`)
- Socket.io를 통한 실시간 브로드캐스팅

---

### 5. 관객 라이브 시청 (`/live/[id]`)
**접근**: 모두 (비로그인 포함, 일부 기능 제한)
**주요 기능**:
- **공연 정보**: 제목, 싱어명, 위치, 시작 시간
- **실시간 시청자 수**: Socket.io 룸 참여자 수 표시
- **라이브 채팅**:
  - 채팅창 열기/닫기
  - 메시지 전송 (닉네임 입력, 비로그인도 가능)
  - 실시간 메시지 수신
- **요청곡**:
  - 곡 제목/아티스트 입력하여 요청
  - (로그인 시 포인트 소모)
- **후원하기**:
  - 포인트 후원 (로그인 필요)
  - Stripe 결제 후 포인트 충전
- **팔로우**: 싱어 팔로우/언팔로우
- **공유하기**: 클립보드에 URL 복사
- **공연 종료 시**: 30초 후 `/explore`로 리다이렉트

**권한**:
- 비로그인: 닉네임 입력 후 채팅/요청곡 가능, 후원 불가
- 로그인(관객): 후원, 팔로우, 포인트 충전 가능
- 로그인(싱어): 자신의 공연이면 대시보드로 리다이렉트

---

### 6. 싱어 프로필 (`/singer/[id]`)
**접근**: 모두 (비로그인 포함)
**주요 기능**:
- **싱어 정보**: 무대 이름, 바이오, 소셜 링크, 인증 뱃지
- **공연 이력**: 진행한 공연 목록
- **팬 수**: 전체 팔로워 수 표시
- **팔로우 버튼**: 팔로우/언팔로우 토글
- **북킹 요청**: 공연 섭외 요청 모달
- **QR 코드**: 싱어 전용 QR (팬 유입용)

**권한**:
- 비로그인: 조회만 가능
- 로그인: 팔로우, 북킹 요청 가능

---

### 7. 가이드 센터 (`/guides`, `/guides/[slug]`)
**접근**: 모두 (비로그인 포함)
**주요 기능**:
- **가이드 목록**: 버스킹 관련 아티클 인덱스
- **검색**: 가이드 제목/내용 검색
- **아티클 상세**: 정적 콘텐츠 (한국어, 1,000자 이상)
- **관련 아티클**: 내부 링크
- **SEO 최적화**: 메타데이터, OG 태그

**권한**:
- 모두 읽기 가능 (로그인 불필요)
- **목적**: Google AdSense 승인용 크롤링 최적화

---

### 8. 법적 페이지 (`/about`, `/privacy`, `/terms`, `/contact`)
**접근**: 모두 (비로그인 포함)
**주요 기능**:
- **About**: 서비스 소개, 연락처
- **Privacy**: 개인정보 처리방침 (Google 광고 쿠키 포함)
- **Terms**: 이용약관
- **Contact**: 문의처 (`support@busking.minibig.pw`)

**권한**:
- 모두 읽기 가능
- **목적**: AdSense 승인 필수 페이지

---

### 9. 인증 관련 (`/sign-in`, `/sign-up`, `/auth/*`)
**접근**: 비로그인 사용자
**주요 기능**:
- **Clerk 인증**: 이메일/구글/깃허브 로그인
- **데모 로그인**: 자동 생성된 데모 계정으로 로그인
- **프로필 생성**: 첫 로그인 시 `Profile` + `Singer` 자동 생성

**권한**:
- Clerk이 서버 사이드에서 인증 처리
- 클라이언트 페이로드로는 인증 불가 (서버 파생 ID만 신뢰)

## API 엔드포인트별 권한

| 엔드포인트 | 메서드 | 접근 권한 | 설명 |
|------------|--------|----------|------|
| `/api/performances` | GET | 모두 | 공연 목록 조회 |
| `/api/performances` | POST | 싱어만 | 공연 생성 (서버 파생 ID) |
| `/api/performances/[id]` | GET | 모두 | 공연 상세 조회 |
| `/api/performances/[id]` | PATCH | 싱어(소유자) | 공연 상태 변경 |
| `/api/singers/[id]` | GET | 모두 | 싱어 프로필 조회 |
| `/api/singers/[id]/follow` | POST | 로그인 사용자 | 팔로우 토글 |
| `/api/song-requests` | POST | 로그인 사용자 | 요청곡 생성 |
| `/api/booking` | POST | 로그인 사용자 | 북킹 요청 |
| `/api/payment` | POST | 로그인 사용자 | Stripe 포인트 결제 |
| `/api/points` | GET/POST | 로그인 사용자 | 포인트 조회/차감 |
| `/api/setlist` | POST/PATCH | 싱어(소유자) | 세트리스트 관리 |

**보안 규칙**:
- 모든 쓰기 작업: Clerk 서버 사이드 `auth()`로 ID 파생
- 소유권 확인: Prisma `Performance.singerId`와 `userId` 비교
- GET 핸들러: DB 쓰기 금지 (읽기 전용 라이프사이클 리졸버 사용)

## Realtime (Socket.io) 권한

| 이벤트 | 발신자 | 권한 |
|---------|--------|------|
| `join_room` | 모두 | 룸 참여 (공연 ID) |
| `send_message` | 모두 | 메시지 전송 (닉네임/로그인) |
| `song_request` | 로그인 사용자 | 요청곡 전송 |
| `open_chat` | 싱어(소유자) | 채팅창 열기 (포인트 차감, API 먼저) |
| `performance_ended` | 싱어(소유자) | 공연 종료 브로드캐스트 |
| `system_alert` | 싱어(소유자) | 시스템 알림 전송 |
| `donation_received` | 싱어(소유자) | 후원 알림 브로드캐스트 |

**보안 규칙**:
- Socket 레이어는 **비권한적 전송자** (transport only)
- 권한 필요 이벤트: Next.js API로 먼저 인증 → 브로드캐스트
- Redis: 캐시/히스토리만, 권한 소스 아님
