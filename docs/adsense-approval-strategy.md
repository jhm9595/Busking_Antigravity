# 구글 애드센스 승인 전략 (PM 공유용)

**로그인이 필요한 폐쇄형 사이트**는 구글 애드센스 승인 과정에서 **"콘텐츠 없음"**이나 **"사이트 검토 불가"** 사유로 거절될 확률이 매우 높습니다. 애드센스 심사 봇(크롤러)이 로그인 벽에 막혀 내부 콘텐츠를 제대로 수집하고 확인할 수 없기 때문입니다.

이를 해결하고 승인 확률을 대폭 높이기 위한 단계별 전략과 액션 아이템을 정리했습니다. PM님께서는 아래 체크리스트를 업무와 백로그(Backlog)에 반영해 주시기 바랍니다.

---

## 1. 승인 전: "공개 콘텐츠" 확보 (가장 중요)
애드센스 계정이 최종 승인(활성화)되기 전에는 '크롤러 로그인 설정' 메뉴를 사용할 수 없는 경우가 많습니다. 따라서 우선은 **로그인 없이도 볼 수 있는 공개 페이지와 콘텐츠 풀**을 충분히 설계해야 합니다.

- [ ] **랜딩 페이지 및 소개 강화:** 서비스의 가치를 설명하는 고품질의 텍스트 콘텐츠를 메인 화면(Landing)이나 별도의 서비스 소개 페이지에 풍부하게 배치하세요.
- [ ] **공개 게시판/블로그 운영:** 사이트의 핵심 기능(공연 관람 등)은 로그인 후 이용하더라도, 고객과 소통할 수 있는 관련 정보, 공지사항, 버스킹 팁 등을 공유하는 **공개 게시판(블로그 형태)**을 만드세요. **글을 20개 이상 (글자 수 1,000자 이상)** 꾸준히 작성해 두는 것이 필수적입니다.
- [ ] **필수 페이지 구축 (Must-have):** 개인정보처리방침(Privacy Policy), 이용약관(Terms of Service), 문의하기(Contact) 페이지는 로그인 없이 누구나 접근 가능해야 하며, 이는 승인의 절대적인 기본 조건입니다.

---

## 2. 승인 신청 시: 크롤러 접근 허용 설정
현재 사이트 구조상 필수적으로 로그인 세션이 필요한 페이지(가수 대시보드, 라이브 스트리밍 방 등)를 심사받아야 한다면, 구글 크롤러에게 사이트 내부를 돌아다닐 수 있는 "마스터 키"를 쥐여줘야 합니다.

- [ ] **테스트 계정 생성:** 구글 크롤러가 전용으로 사용할 테스트 아이디와 비밀번호를 생성합니다.
- [ ] **애드센스 크롤러 로그인 정보 추가:**
  1. 애드센스 대시보드 접속
  2. `계정` > `액세스 및 승인` > `크롤러 액세스` 로 이동
  3. **로그인 정보 추가**를 클릭하여 로그인 URL(`/sign-in`), 아이디, 비밀번호를 정확하게 입력합니다.
- [ ] **Search Console 소유권 인증:** 해당 사이트의 도메인이 본인 소유임을 구글 서치 콘솔(Google Search Console)에서 사전에 인증해 두어야 크롤러 액세스 설정이 정상적으로 작동합니다. (DNS 레코드 또는 메타 태그 인증 방식 사용)

---

## 3. 기술적 체크리스트 (Technical SEO)
사이트의 차단 설정으로 인해 크롤러가 튕겨 나가지 않도록 파일들을 점검해야 합니다.

- [ ] **Robots.txt 확인:** 프로젝트 최상단(`public/robots.txt`)에 구글 애드센스 크롤러(`Mediapartners-Google`)의 접근을 차단하고 있지 않은지 반드시 확인하세요.

**[올바른 Robots.txt 설정 예시]**
```text
User-agent: Mediapartners-Google
Allow: /
```
**(주의: `Disallow:` 란에 슬래시(`/`)나 차단 경로가 설정되어 있으면 안 됩니다.)**

---

**PM (프로젝트 매니저) 전달 사항 요약:**
> 위 전략에 따라 **로그인 없이 볼 수 있는 고품질 콘텐츠 페이지 추가 기획(블로그/게시판 등)**을 최우선으로 진행 부탁드리며, 개발팀과 협업하여 **크롤러 테스트 계정 발급 및 robots.txt 설정 리오픈**을 일정에 반영해 주시기 바랍니다.
